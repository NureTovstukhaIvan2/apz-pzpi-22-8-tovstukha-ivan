1 БІЗНЕС-ВИМОГИ

1.1 Передумови

     Сучасні океанаріуми є складними екосистемами, які потребують постійного моніторингу та контролю для підтримання здоров’я морських організмів. Основними викликами, з якими стикаються власники та оператори океанаріумів, є необхідність забезпечення стабільних умов середовища, таких як температура води, рівень кисню і кислотність (pH). Найменші відхилення від нормальних параметрів можуть негативно вплинути на морську флору та фауну, що призводить до фінансових втрат, а також до ризику втрати життя морських мешканців.
     Ідея створення даної програмної системи полягає у розробці інструменту, який автоматизує процеси моніторингу та управління параметрами середовища в океанаріумах, зменшуючи залежність від людського втручання. Така система дозволить своєчасно реагувати на критичні зміни, підвищуючи ефективність управління та зменшуючи ризики для морських організмів.
     Мета розробки продукту — забезпечити операторів океанаріумів сучасною автоматизованою системою управління, яка дозволить їм у режимі реального часу контролювати стан середовища, швидко реагувати на відхилення та ефективно використовувати ресурси. Крім того, система буде корисною для освітніх і наукових закладів, які використовують океанаріуми для досліджень та навчання, оскільки вона забезпечує зручний інструмент для збору, аналізу та візуалізації даних.
     Цей програмний продукт створюється для зменшення навантаження на персонал, оптимізації витрат, підвищення точності моніторингу та надання можливості масштабування системи для океанаріумів різного розміру.

     У підсумку, кінцевий споживач отримає надійну платформу для підтримки життя морських організмів, яка сприяє стабільності екосистеми, покращує якість обслуговування та допомагає уникнути потенційних проблем у майбутньому.
1.2 Бізнес-можливості

     Програмна система орієнтована на ринок комерційних океанаріумів, зоопарків, науково-дослідних центрів, освітніх закладів та приватних акваріумів великого масштабу. Ринок автоматизованих систем моніторингу та управління є перспективним через зростання популярності інтеграції IoT- рішень у сферу обслуговування та екологічного управління. Оглянемо подібні системи, та порівняємо з нашою:
   1) Seneye Reef – це система для моніторингу параметрів води в акваріумах та океанаріумах. Вона вимірює основні параметри, такі як рівень pH, аміак, температура та освітлення. Система забезпечує підключення до мобільного додатку та хмарних сервісів, де користувачі можуть переглядати дані в режимі реального часу.
Переваги:
* простота встановлення та використання;
* інтеграція з мобільними додатками;
* можливість збереження історії даних у хмарі. Недоліки:
* відсутність автоматизованого управління пристроями
 * орієнтований на невеликі акваріуми, не масштабований для великих океанаріумів;
* обмежена кількість параметрів моніторингу.

   2) Apex Fusion – це комплексна система управління акваріумами, яка дозволяє моніторити та керувати параметрами, такими як температура, солоність, рівень pH і кисню. Система забезпечує інтеграцію з хмарними сервісами для зручного управління через додаток або вебінтерфейс.
Переваги:
* висока точність моніторингу параметрів;
* інтеграція з численними сторонніми пристроями;
* хмарне управління та доступ до даних у будь-який час. Недоліки:
* висока вартість системи;
* складність установки та інтеграції, потреба в професійній допомозі;
* залежність від підключення до хмари для повної функціональності.
   3) GHL Profilux – це висококласна система контролю для акваріумів, яка пропонує широкий спектр можливостей для моніторингу та управління, включаючи освітлення, температуру, хімічний склад води тощо.
Переваги:
* розширений набір функцій та сенсорів;
* можливість створення кастомних сценаріїв управління;
* підтримка великої кількості модулів для розширення. Недоліки:
* надмірна складність для базових потреб;
* висока ціна, що робить систему недоступною для невеликих або середніх океанаріумів;
* обмежена інтеграція з іншими платформами.

     Основними перевагами нашого продукту є його здатність повністю автоматизувати процес управління середовищем океанаріумів, забезпечуючи не лише моніторинг параметрів, але й контроль пристроїв, які коригують ці параметри. Система розроблена з урахуванням гнучкості та масштабованості, що дозволяє адаптувати її до будь-якого розміру океанаріуму, а також інтегрувати додаткові сенсори для специфічних потреб користувачів.
     Продукт зберігає свою економічну привабливість, залишаючись доступним для невеликих і середніх установ без компромісів щодо основної функціональності. Простота інтеграції з іншими IoT-системами завдяки використанню стандартних протоколів, таких як MQTT, дозволяє значно розширити можливості системи.
     Основне призначення системи полягає у створенні стабільного середовища шляхом автоматизації процесів, які раніше вимагали значного людського втручання. Вона забезпечує безперервний контроль параметрів води через веб-інтерфейс та мобільний додаток, що дозволяє користувачам моніторити стан акваріумів з будь-якого місця. Система підтримує мультимовні інтерфейси, які дають змогу візуалізувати поточні показники в режимі реального часу, аналізувати історичні дані, отримувати повідомлення про аварійні ситуації та налаштовувати автоматичні сценарії реагування. Завдяки підтримці масштабованості, продукт можна адаптувати як для невеликих приватних акваріумів, так і для великих океанаріумів, при цьому система легко розширюється за рахунок додавання нових сенсорів або пристроїв у разі зростання потреб користувача.
     Система працює автономно, забезпечуючи стабільність навіть без постійного доступу до хмарних сервісів, що робить її зручною для клієнтів у віддалених регіонах або при необхідності локальної роботи. Ми орієнтуємося на потреби кожного клієнта, створюючи рішення без зайвих функцій, що дозволяє сконцентруватися на основних завданнях.
     
     Ці характеристики роблять наш продукт оптимальним вибором для широкого кола користувачів, включаючи великі комерційні океанаріуми, науково-дослідні організації та ентузіастів, які прагнуть підтримувати ідеальні умови для своїх акваріумів.

2 КОНЦЕПЦІЯ РІШЕННЯ

2.1 Окреслення концепції

     Дана програмна система - це комплексне рішення, яке поєднує автоматизований моніторинг, управління пристроями та аналітику для підтримання оптимальних умов життя морських організмів.
     Ця система розроблена для задоволення потреб операторів океанаріумів, дослідницьких установ, освітніх закладів та комерційних акваріумів будь- якого масштабу.
     Основне призначення системи полягає у створенні стабільного середовища шляхом автоматизації процесів, які раніше вимагали значного людського втручання.
     Система забезпечує безперервний контроль параметрів води, таких як температура, рівень кисню та кислотність (pH), а також автоматично вмикає чи вимикає відповідні пристрої (терморегулятор, аератор, контролер pH) у разі відхилення параметрів від встановлених норм. Це знижує ризики для морської екосистеми та підвищує ефективність роботи персоналу.
     Система надає зручний веб-інтерфейс для оператора, який дозволяє візуалізувати поточні показники в режимі реального часу, аналізувати історичні дані, отримувати повідомлення про аварійні ситуації та налаштовувати автоматичні сценарії реагування.
     Вбудована підтримка IoT-рішень та MQTT-протоколу забезпечує легку інтеграцію з існуючими або новими пристроями та сенсорами.
     
     Продукт підтримує масштабованість, дозволяючи адаптувати систему як для невеликих приватних акваріумів, так і для великих океанаріумів із розвиненою інфраструктурою. Система також дозволяє додавати нові сенсори або пристрої в міру зростання потреб клієнта.
     Ця система допомагає не лише забезпечувати комфортні умови для морських мешканців, але й оптимізувати витрати енергії та ресурси, а також знижувати операційні витрати. Продукт надає власникам океанаріумів впевненість у стабільності середовища та можливість зосередитися на стратегічних завданнях замість ручного контролю процесів.
     Таким чином, програмна система для автоматизації та моніторингу акваріумів є сучасним інструментом, який забезпечує баланс між екологічною відповідальністю, технологічним прогресом та економічною ефективністю, відкриваючи нові можливості для вдосконалення управління морськими екосистемами.
2.2 Головна функціональність

MF-1. Автоматичний моніторинг показників акваріуму
     Система постійно моніторить основні показники акваріуму, такі як температура води, рівень кисню та рівень pH.
MF-2. Автоматична корекція параметрів
     При виявленні відхилення будь-якого параметра від встановлених норм (температура, кисень, pH), система автоматично вмикає відповідний пристрій для повернення параметра до норми.
MF-3. Інтеграція з IoT пристроями
     Система використовує IoT-пристрої (ESP32) для збирання даних із сенсорів та управління підключеними пристроями. Завдяки цьому забезпечується автоматизований моніторинг параметрів акваріума та їх корекція в реальному часі.
     
MF-4. Візуалізація показників на OLED-дисплеї
     Стан акваріума та підключених пристроїв (ON/OFF) відображається на OLED-дисплеї у зручному форматі. Це дозволяє власнику акваріума отримувати основну інформацію без доступу до зовнішніх інтерфейсів.
MF-5. Ведення журналу дій
     Система фіксує всі зміни параметрів сенсорів та станів пристроїв у базі даних (таблиця Logs). Кожна дія автоматизації (вмикання/вимикання пристроїв) супроводжується логом, що дозволяє відстежувати історію подій.
MF-6. Інтерактивний REST API
     Система надає REST API, який дозволяє користувачам взаємодіяти з акваріумом через зовнішні додатки або платформи.
MF-7. Інтеграція з публічним MQTT-брокером
     Система використовує публічний MQTT-брокер (Mosquitto), що забезпечує стабільну передачу даних у реальному часі. Це дозволяє користувачам підключатися до системи без потреби у налаштуванні власного серверного середовища для обміну повідомленнями.
MF-8. Автоматичне оновлення даних сенсорів у базі даних
     Система автоматично оновлює значення сенсорів у базі даних після кожного моніторингу, забезпечуючи точність інформації для подальшого аналізу або інтеграції з іншими системами.
MF-9. Повідомлення у реальному часі
     Система генерує MQTT-повідомлення про стан сенсорів та пристроїв у реальному часі, що дозволяє споживачам використовувати ці дані для віддаленого моніторингу або інтеграції з іншими IoT-додатками.
MF-10. Гнучка архітектура
     Система побудована на базі Node.js та MQTT, що забезпечує високу масштабованість і дозволяє додавати нові сенсори, пристрої чи функції без значних змін у коді.
     
MF-11. Мобільний додаток Android
     Система включає повнофункціональний мобільний додаток для Android, який підтримує як англійську, так і українську мову, синхронізується з веб-системою, надсилає push-повідомлення у разі критичних змін параметрів та здатен працювати в офлайн-режимі з подальшою автоматичною синхронізацією даних.
MF-12. Локалізовані звіти
     Система генерує звіти у форматі, адаптованому під обрану мову інтерфейсу, з урахуванням регіональних налаштувань (одиниці виміру, формати дат тощо).
MF-13. Багатомовна підтримка клієнтів
     Система включає повністю локалізовані елементи, зокрема інтерфейс адміністратора, довідкові матеріали, повідомлення системи, а також електронні листи та сповіщення, що забезпечує зручність використання для користувачів різними мовами.
MF-14. Автоматичне визначення мови
     Система автоматично визначає мову користувача на основі налаштувань браузера або мобільного пристрою з можливістю ручного перевибору.
MF-15. Локалізована документація
     Повна документація системи доступна англійською та українською мовами як у веб-інтерфейсі, так і в мобільному додатку.

3 РАМКИ ТА ОБМЕЖЕННЯ

3.1 Рамки первинного продукту

     Перший реліз програмної системи AquaSense представляє собою комплексне рішення для моніторингу та автоматизації акваріумних систем, що поєднує сучасні веб-технології, IoT-пристрої та мобільний додаток. Система забезпечує базовий функціонал для ефективного управління параметрами водного середовища, включаючи автоматичну корекцію температури, рівня кисню та кислотності через інтегровані пристрої. Центральним компонентом є веб-додаток на React з повноцінною підтримкою англійської та української мов, що дозволяє користувачам моніторити стан акваріумів у реальному часі з будь-якого пристрою. Мобільний додаток для Android синхронізується з основною системою, забезпечуючи доступ до всіх функцій зі смартфона, включаючи push-сповіщення про критичні зміни параметрів.
     Основною метою даної системи є створення базової версії, яка дозволить операторам океанаріумів ефективно відстежувати стан середовища в реальному часі, автоматично коригувати параметри води та управляти обладнанням. Вона повинна забезпечити зручний інтерфейс для моніторингу ключових параметрів акваріумів, таких як температура, рівень кисню та pH, а також автоматизувати управління пристроями, такими як терморегулятори, аератори та контролери pH, для підтримки стабільних умов у воді. Завдяки автоматизації та інтеграції різних пристроїв, система також підвищить енергоефективність та оптимізує використання ресурсів, забезпечуючи комфорт для мешканців акваріумів.
Система  включає  кілька  основних  функціональних  компонентів.
Моніторинг параметрів середовища здійснюється через IoT-сенсори, які автоматично записують дані в базу. У разі відхилення будь-якого з параметрів від встановлених норм система активує відповідні пристрої для автоматичної корекції значень. Дані з сенсорів і стан пристроїв відображаються на OLED- дисплеї, а всі зміни реєструються в журналах подій. Логи доступні для подальшого аналізу через REST API. Також система інтегрована з протоколом MQTT, що забезпечує ефективну комунікацію між IoT-пристроями та сервером.
     Серверна частина, побудована на Node.js та Express, обробляє дані з сенсорів ESP32, які передають інформацію через MQTT-протокол. Система автоматично аналізує отримані показники та активує відповідні пристрої (терморегулятори, аератори, pH-контролери) при виявленні відхилень від норми. Всі зміни фіксуються в базі даних MySQL і доступні для перегляду через REST API. Ключовою особливістю є повноцінна багатомовна підтримка, що включає не тільки інтерфейс користувача, а й системні повідомлення, документацію та звіти, адаптовані під вибрану мову та регіональні налаштування.  
     IoT-пристрій, який базується на платформі ESP32, відповідає за збір даних із сенсорів та передавання їх на сервер через MQTT. Окрім того, цей пристрій виконує команди для включення або вимикання пристроїв відповідно до вказівок сервера. Всі поточні значення параметрів і статус пристроїв відображаються на OLED-дисплеї, а індикація стану пристроїв здійснюється через світлодіоди.
     База даних складається з кількох таблиць, що зберігають дані про користувачів, сенсори, пристрої, журнали подій та історичні дані. Історія змін параметрів та стану пристроїв зберігається для подальшого аналізу і моніторингу.
     Бізнес-логіка системи включає кілька ключових процесів. IoT-пристрій регулярно зчитує показники сенсорів і передає їх на сервер. Сервер перевіряє ці значення на відповідність заданим межам і, якщо параметр виходить за межі норми, надсилає команду на включення пристрою для корекції. Після стабілізації параметра пристрій вимикається. Логування кожної зміни стану пристроїв та параметрів дозволяє здійснювати подальший аналіз подій через API. Адміністратор має можливість додавати або редагувати пристрої, сенсори, а також коригувати допустимі межі параметрів через REST API.
     Система також передбачає захист даних. Всі дані користувачів зберігаються в зашифрованому вигляді, а доступ до API обмежується через систему авторизації. Масштабованість системи дозволяє інтегрувати нові сенсори і пристрої без потреби в суттєвих змінах у структурі системи.
     В першій версії система забезпечує основну функціональність, необхідну для ефективного моніторингу та управління акваріумами, включаючи автоматичну корекцію параметрів, реєстрацію змін у логах та інтеграцію з іншими системами. Це стане основою для подальшого розвитку та вдосконалення можливостей системи AquaSense.
3.2 Рамки наступних випусків

     У наступних версіях системи передбачено розширення функціональності з метою підвищення зручності використання, інтеграції з іншими платформами, розширення можливостей моніторингу та створення мобільного клієнта.
Реалізація веб-інтерфейсу для управління та моніторингу:

* перегляд поточних значень параметрів у реальному часі;
* налаштування діапазонів допустимих значень для сенсорів;
* перегляд історичних даних у вигляді таблиць;
* керування пристроями вручну (вмикати/вимикати);
* інтеграція з REST API для доступу до даних і управління пристроями.
Розширення функціональності логування:

* додавання можливості автоматичного експорту логів у форматах CSV або Excel (.xlsx) для подальшого аналізу;
* система автоматичних звітів, які можуть надсилатися на електронну пошту операторів із зведенням за обраний період.
Додаткові сенсори:

* інтеграція нових типів сенсорів для розширення функціональності, таких як сенсори солоності (для морських океанаріумів) та сенсори рівня води;

* відображення нових параметрів у веб-інтерфейсі та на OLED- дисплеї.
Покращення алгоритмів управління:

* додавання можливості комбінованої корекції параметрів, коли кілька пристроїв працюють одночасно для досягнення оптимальних умов;
* оптимізація алгоритмів для більш ефективного використання енергії та ресурсів.
Мультиакваріумна підтримка:

* додавання можливості управління кількома акваріумами одночасно з використанням єдиного веб-інтерфейсу;
* розмежування даних для кожного акваріума.

Реалізація мобільного клієнта на Android:

* перегляд поточних значень сенсорів у реальному часі;
* сповіщення про критичні зміни параметрів акваріуму;
* дистанційне керування пристроями через мобільний інтерфейс;
* можливість налаштувати діапазони параметрів для сенсорів та сповіщень;
* можливість	переглядати	історичні	дані	та	таблиці	прямо	в мобільному додатку.
     Основною метою подальших версій є зробити систему більш зручною, інтерактивною та масштабованою, зберігаючи її простоту та надійність. Зосередження на додаткових інструментах управління та розширеній аналітиці допоможе залучити більше користувачів і забезпечити конкурентоспроможність продукту на ринку.

4.3 Робоче середовище

Технології та програмні засоби:

* платформи: Node.js (сервер), Express.js (REST API), MySQL (база даних), MQTT (Mosquitto для комунікації);
* бібліотеки: bcrypt.js (авторизація), Swagger UI (документування API), PubSubClient (MQTT-клієнт для ESP32), Adafruit SSD1306 (OLED-дисплей), i18next;
* мови: JavaScript (сервер), C++ (ESP32), Kotlin/Java (мобільний додаток);
* обладнання IoT: ESP32, сенсори (температура, кисень, pH), OLED-дисплей, світлодіоди для індикації.
Вимоги до середовища:

* програмне забезпечення: MySQL 5.7+, Node.js 14+, Arduino IDE;
* обладнання: ESP32, сенсори, сервер (2 ядра CPU, 2 ГБ RAM, 50 ГБ дискового простору), Android-пристрій (версія 8.0+) для мобільного клієнта;
* ОС: Windows, Linux, macOS, Android. Основні вимоги:
* доступність: REST API та локальний доступ через OLED- дисплей;
* цілісність: захист даних сенсорів у базі через транзакції MySQL, збереження логів для подальшого аналізу.
* продуктивність: реакція системи на відхилення параметрів у межах 1 секунди, відповідь API < 200 мс;
